  笔记本连 GitHub · 不用手机装 GitHub App
═══════════════════════════════════════════════════════

【结论】
  你手机不用装任何 GitHub 软件。
  所有登录、发布都在「这台笔记本」完成。

【当前网络情况（已检测）】
  · 本机 → github.com 的 443 端口：经常连不上（不是你不会用，是线路问题）
  · 本机 → gitee.com：可以连上

  所以有两条路，任选其一：

───────────────────────────────────────────────────────
路线 A · 继续用现在的 GitHub 链接（学员链接不变）
───────────────────────────────────────────────────────

  学员链接仍是：
  https://saivenwang-byte.github.io/XiaoWangXueRiyu/index.html?v=246

  你要「更新内容」上传到 GitHub，需要笔记本能访问 GitHub，任选：

  1) 开 VPN / 代理（Clash 等）后，双击：
     「笔记本推送到GitHub.bat」

  2) VPN 能开浏览器但命令行仍失败时，双击：
     「GitHub网页上传-不用Git命令.bat」
     用笔记本浏览器打开 GitHub 网页，手动上传改过的文件。

  登录方式（只需笔记本浏览器）：
  · 双击 bat 后按提示，在「本机 Edge/Chrome」打开 github.com 登录
  · 或输入一次性验证码 —— 都在笔记本上完成，与手机 App 无关

───────────────────────────────────────────────────────
路线 B · 改用 Gitee（国内网络最稳，推荐备份）
───────────────────────────────────────────────────────

  双击：「笔记本发布到Gitee.bat」
  → 用 gitee.com 账号（笔记本浏览器注册）
  → 推送代码 → 开启 Gitee Pages → 得到新的 https 链接
  → 把新链接发给学员（与 GitHub 二选一，或暂时两条都能用）

  详见 bat 里每一步说明。

───────────────────────────────────────────────────────
【不要做的事】
───────────────────────────────────────────────────────

  × 在手机上装 GitHub App（不需要）
  × 指望不连 VPN 时 git push 一定能成功（本机已测 443 不通）
  × 只改电脑文件却不发布 —— 学员看到的还是旧网页
