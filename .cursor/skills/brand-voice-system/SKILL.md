# Brand Voice System

## 核心定位

把风格从个人感觉变成可传递的写作规则。

## 触发场景

- 用户明确说要做与本 skill 描述相关的任务
- 用户提供素材、数据、链接、会议记录、代码或业务问题，需要整理成可执行交付物
- 用户想减少来回沟通，希望 AI 先给出结构化方案和下一步动作

## 工作流

### Step 1: 明确目标和约束

先提取用户真正想要的结果、受众、使用场景、输入材料、截止时间和不可触碰的边界。信息不足但不影响启动时，先写明合理假设继续推进。

### Step 2: 拆解输入材料

把原始材料拆成事实、判断、待验证假设和可执行动作。不要把未经验证的推测包装成结论。

### Step 3: 生成一版可交付结果

围绕用户目标生成完整初稿或分析结果。优先给可直接使用的内容，其次给解释。

### Step 4: 自检和迭代

按本任务的质量标准检查：是否具体、是否可执行、是否有证据、是否遗漏关键风险。发现问题时直接修正，不只提示用户自己改。

## 输出格式

```text
目标：...
关键输入：...
主要结论/产出：...
声纹规则、禁用词表、示例改写、审核清单：...
风险与待确认：...
下一步：...
```

## Gotchas

1. **不要输出泛泛建议**
   解释：学员需要能马上使用的交付物。每条建议都要对应具体动作、判断依据或可检查结果。

2. **不要假装已经验证外部事实**
   解释：涉及市场、金融、工具版本、平台规则和新闻时，要说明来源或标注待验证。

3. **不要无限扩写**
   解释：先交付最小完整版本，再根据用户反馈加深。复杂内容可以拆成清单、表格和分阶段计划。
