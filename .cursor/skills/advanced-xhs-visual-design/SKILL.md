# 高级图文设计

把内容转成可传播的高质感图文卡片；当用户只要生图 prompt 时，输出可直接投喂模型的高级提示词。

## 核心定位

- 负责：选题拆解、卡片叙事、版式系统、色彩字体、单页构图、出图 prompt、HTML/CSS 设计稿规范。
- 同时负责：为封面、海报、信息图、人像、UI 截图、角色图、叙事概念图生成高级生图 prompt。
- 不负责：自动发布小红书；需要发布时交给发布类 skill。
- 不负责：真实生图；需要出图时交给生图类 skill。
- 默认先交付可执行结果，不输出理论说明。

## 先判断任务分支

### A. 图文卡片分支

用户要“小红书图文 / 公众号图文 / 卡片 / 图文设计 / 把文章做成图 / 复刻版式”时，走完整图文方案。

默认输出：

1. 视觉方向：一句话定义风格与记忆点。
2. 卡片脚本：6-9 张卡，每张包含目的、标题、画面结构、强调色。
3. 设计系统：尺寸、背景、字体、颜色、组件、间距。
4. 逐页出图 prompt：每张图一条可直接投喂生图模型或交给 HTML/CSS 渲染的描述。
5. 质检清单：确保像设计作品，不像 AI 模板。

### B. Prompt-only 分支

用户只要“写 prompt / 高级画图提示词 / 超级提示词 / 仿写 prompt / 只要一条提示词”时，不强行做整套图文。

默认输出：

1. 结论：推荐哪版最适合当前用途。
2. Prompt 1｜稳妥版：最容易出可用图。
3. Prompt 2｜传播版：更抓眼，更适合封面传播。
4. Prompt 3｜高概念版：更强视觉记忆点和设计感。
5. 建议参数：比例、风格关键词、负面约束。

如果用户明确只要 1 条，就只给 1 条。

## 工作流

### 1. 判断内容类型

- 产品发布 / 功能更新：走 release notes + terminal dashboard。
- 行业复盘 / 公司战略：走时间线 + 战略地图 + 数据卡。
- 工具评测 / 教程：走问题冲击 + 操作流程 + 对比结果。
- 投资 / 商业分析：走核心判断 + 数据证据 + 机制拆解 + 风险。
- 观点型内容：走强观点封面 + 证据模块 + 结论金句。
- 纯生图 prompt：判断为摄影人像、海报/封面、信息图、UI/截图、角色图、叙事概念图。

### 2. 先做叙事，不先做装饰

图文卡片必须有清楚节奏：

- 第 1 张：封面，负责停留率。
- 第 2-3 张：核心变化/核心机制，讲清“发生了什么”。
- 第 4-6 张：证据、数据、案例、时间线，建立可信度。
- 第 7-8 张：战略判断/影响，形成传播观点。
- 最后一张：一句话收束，负责收藏和转发。

每张图只承载一个主要判断。不要把文章段落搬进图里。

### 3. 选择视觉母版

默认优先使用 **暗黑科技发布页**：

- 黑色/近黑背景，有细噪声、扫描线、暗纹或微弱网格。
- 荧光绿作为主强调色；黄、青、粉只作数据和标签辅助色。
- 超大中文黑体标题；英文、命令行、标签使用等宽字体。
- 终端窗口、代码块、时间线、指标卡、发光边框作为组件。

详细规则读取：`references/tech-xhs-design-system.md`。

### 4. 生成高级 prompt

所有 prompt 必须覆盖：

- 媒介/画面类型：film photography、poster、infographic、UI screenshot、editorial card。
- 主体与场景：谁、什么物、什么场景、什么身份或时代。
- 镜头与构图：比例、视角、留白、主视觉支点、视觉动线。
- 光线系统：自然光、闪光、霓虹、rim light、window light、屏幕光。
- 材质与细节：纸张、噪声、皮肤、织物、玻璃反射、UI 间距。
- 色彩策略：主色 + 强调色，不乱堆颜色。
- 情绪与叙事：让画面有明确气质和故事线索。
- 文字/版式约束：中文可读、标题位置、信息层级。
- 负面约束：主动排除廉价、乱码、过度发光、拥挤、模板感。

先定媒介，再定主体；先给一个强视觉支点，再补次级证据。抽象审美词后面必须跟可见证据词。

### 5. 做事实与文字保护

- 涉及公司、产品、融资、收入、增长、日期、模型版本、市场数据时，不确定就标为“待核验”。
- 用户要求真实发布时，必须先核验来源。
- 不要把示意数字伪装成真实数据。
- 生图 prompt 可以写 `mock data / illustrative labels`，正式发布稿必须换成已核验信息。
- 中文信息图、中文海报、中文界面，必须写明“所有可见文字使用简体中文且清晰可读”。
- 复杂中文排版优先用 HTML/CSS/设计工具渲染，生图只负责背景和装饰。

## 质检清单

- 是否一眼能看出主判断或主视觉支点。
- 是否每页只讲一个主要信息。
- 是否有明确比例、构图、光线、材质、色彩、负面约束。
- 中文是否足够短、足够大、足够可读。
- 荧光色是否克制，是否避免廉价赛博朋克。
- 数据是否标明真实/待核验/示意。
- Prompt 是否能直接复制投喂，而不是一堆解释。

## Gotchas

1. 不要只复刻表面。高级感来自信息层级、留白、组件秩序和文字比例，不是黑底绿字。
2. 不要过度赛博朋克。满屏霓虹、发光过重、机械图标乱飞会立刻廉价。
3. 不要让生图模型自由写长中文。中文必须短、少、明确。
4. 不要编数据。未核验数据必须显式标注。
5. 不要每张都做封面。正文页要让信息系统说话。
6. 不要只堆风格词。必须写构图、光线、材质和关系。
7. 不要把中文版硬翻成英文版。English version 应该改写成更适合模型理解的投喂稿。
8. 不要在 prompt 里加入互相冲突的要求，例如“极简”同时“堆满细节”。
