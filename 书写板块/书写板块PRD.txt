一、产品概述
1.1 板块定义
“书写”板块（日文名：書く）是本小程序继“课文学习”“口语对话”“复习测试”之后的第四功能板块。它专注于日语书写能力的培养，从假名入门到短句听写，与标日课本内容一一对应，帮助小朋友在课后巩固书写技能。

1.2 核心价值
价值点	说明
补齐能力闭环	听说读写四大技能完整覆盖
溯源课本	每个书写练习都能回溯到标日课本的具体出处
即时反馈	笔顺检查、字形评分，即时纠正错误
联动学习	书写内容与语法卡片、对话练习、错题复习深度打通
1.3 目标用户
角色	核心诉求
小朋友（直接使用者）	写对假名、记住汉字、能写出完整的日语句子
家长（监督者）	看到孩子的书写进步，了解薄弱字形
1.4 使用场景
小朋友在学校学完标日某一课 → 打开小程序 → 进入“書く”板块 → 选择对应课程 → 完成单词书写或听写练习 → 即时获得评分 → 错字自动加入复习列表。

3.3 书写区域设计
元素	设计
田字格	浅灰色虚线，2×2分割，辅助用户掌握字形比例
笔迹	可变颜色（默认深灰 #333，可切换为毛笔黑/铅笔灰）
笔触粗细	三档可调（细/中/粗）
橡皮擦	点按擦除、全部清除
参考线	描红模式下显示灰色底字
3.4 评分机制
评分维度	权重	说明
笔画完整性	40%	是否缺少或多出笔画
笔顺正确性	30%	笔画顺序是否正确
字形比例	20%	各部分比例是否协调
整体美观	10%	线条流畅度、整洁度
评分等级：

★★★（90-100分）：完美

★★☆（70-89分）：基本正确，有小瑕疵

★☆☆（<70分）：需要练习，自动加入复习列表

3.5 错字复习
评分低于★★☆的书写自动进入「復習リスト」

复习提醒规则：第1天、第3天、第7天

复习时重新书写，评分达标后从列表移除

3.6 听写模式（L3专属）
步骤	操作
1	播放课文原音（可调节语速：0.5x / 0.75x / 1x）
2	用户在书写区域写出听到的内容
3	提交后与原文比对，逐字标红错误
4	显示正确写法 + 简短说明
四、与其他板块的联动
联动方向	具体实现
与语法卡片联动	书写单词时，点击「📖」可弹出该词对应的语法节点十秒迷你卡片
与对话练习联动	听写内容来自对话练习中的句子，已缓存音频直接调用
与测试联动	书写错误自动加入错题复习列表，纳入三日提醒
与学习看板联动	「マイページ」中展示本周书写完成度、薄弱字形TOP3
与一课三关联动	书写完成纳入课程总进度（作为第四关或选做关）
与语音板块联动	假名书写可直接点击播放标准发音，使用已缓存的本地语音包

六、技术方案
6.1 核心依赖
功能	实现方案
书写画布	HTML5 Canvas 或 微信小程序 Canvas API
笔顺动画	SVG 路径动画
评分引擎	本地算法：Canvas笔迹 → 与模板路径比对
音频播放	复用已有本地缓存的语音文件
数据存储	书写进度、评分、错字列表存入 wx.setStorage
6.2 性能要求
指标	要求
画布响应延迟	<50ms
评分计算时间	<200ms
笔顺动画加载	<1s
6.3 兼容性
微信小程序 Canvas 2D API

支持 iPhone SE 至 Pro Max 屏幕尺寸

离线可用（书写评分引擎本地运行）
