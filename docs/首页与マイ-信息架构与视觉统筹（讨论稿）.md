## 0. 总原则（已写入 `PRODUCT-PM-全局导演手册.md` §4.2）

### 0.1 单元手风琴（原则性问题 · 必须遵守）

**课文首页与「我的」页采用同一规则：任意时刻最多展开 1 个单元。**

- 点开第 6 单元 → 展开；再点第 5 单元 → **第 6 单元自动收起**。  
- 目的：避免手机屏幕变成「很长的一条」需要大量上下滑动。  
- 实现：`details` + `toggle` 监听，与 `journey-home.js` / `app.js` `bindMeUnitAccordion` 一致。

---

## 0.2 总原则（拍板共识）

| 原则 | 说明 |
|------|------|
| **课文 = 行动** | 选单元、进课、学四关、看奖励图；不重复「档案统计」 |
| **我的 = 档案** | 学了多少天/多久、全册进度、笔记、督导；不重复「选课入口」 |
| **通关 = 一套符号** | 课内四关、课次四星、单元四星/条带、全册彩蛋 — 同一套「星 + 灰彩图」语言 |
| **日语优先** | 折叠行主标题用日语大字；中文场景名小字、在下方 |
| **少数字、多图形** | 能用手风琴 + 星 + 进度条表达的，不写 `4/4 课` 这类开发语言 |

## 1. 课文 vs 我的 — 必须分开的两页

### 1.1 现状问题

- 「我的」曾用 **24 课平铺列表 + 四星 + 通関**，与首页 **单元折叠 + 课次卡片** 信息重复。
- 用户感知：两个 Tab 像同一张表的两种视图。

### 1.2 目标分工（建议拍板）

```text
┌─────────────────────────────────────────────────────────────┐
│  课文（首页）          │  我的（マイ）                        │
├──────────────────────┼──────────────────────────────────────┤
│  地图 + 00 入門       │  累计：学习天数 · 学习时长（估）      │
│  六单元折叠手风琴     │  六单元折叠：仅「单元级」进度摘要       │
│  展开 → 4 课入口      │  展开 → 4 课档案行（星/弱项/进课）    │
│  课次右侧：课彩蛋缩略  │  不显示与首页同构的大地图              │
│  单元行尾：单元条预览  │  笔记入口：课 / 单元 / 全册笔记本       │
│  👁 只看已通关条带图  │  漫画收藏册、错题/にがて、设置         │
└──────────────────────┴──────────────────────────────────────┘
```

**首页不出现**：累计天数长文、全册笔记列表、督导小作文。  
**我的不出现**：与首页同尺寸的地图、重复的「点课即学」主列表（可保留「继续学」一条快捷）。

### 1.3 关于 Q4「我的不要地图？」（产品方澄清 · 已理解）

此前表述 **不是**「我的页里绝对不能出现任何地图图形」，而是：

- **课文** = 学习入口（地图 + 选课 + 四关 + 👁 看条带）。  
- **我的** = **个人学习档案**（天数、时长、进度、笔记、督导），导航与信息重心都要围绕「我」的积累，而不是再摆一张和课文一样的「学习地图」。

「我的」可以有 **进度条 / 四星 / 折叠单元名**，但不要做成课文的克隆页。V1 已按此拆分。

## 2. 首页 · 折叠单元行（一行里的元素统筹）

### 2.1 当前行内元素（实现现状）

从左到右大致为：

`▾ 箭头` · `第N单元（日）` / `场景中文` · `4/4 课` · `👁或▦` · `★ 单元通関` 文案

问题：**箭头、数字进度、眼睛、文字「通关」、课内四星** 语义重叠，且视觉权重不统一。

### 2.2 建议的统一符号表（全 App）

| 符号 | 含义 | 出现位置 | 尺寸建议 |
|------|------|----------|----------|
| **▾ / ▸** | 折叠/展开（唯一折叠控件） | 单元行最左 | 16×16px 触控区 44px |
| **★ ★ ★ ★**（课） | 单词·会話·文法·テスト 四关 | 课次行、我的课档案 | 海星组件已有 |
| **★★★★**（单元） | 本单元 4 课均四关满 | 单元行右侧（替代「单元通関」四字） | 与课星同族、略大或金色描边 |
| **👁** | 仅「已单元通关 → 看 L2 四格条带」 | **行最右**，四星左侧 | 24px 图标按钮 |
| **课缩略图** | L1 课彩蛋；灰=未通关 / 彩=已通关 | 课次行右侧 | 现有 egg thumb |
| **▦**（可选） | 与 👁 二选一：条带缩略而非眼睛 | 开发/满配保留 | 验收用 |

**拍板建议**：

- **去掉** 行内 `4/4 课` 文案（见 §3）。
- **去掉** 行内汉字「单元通関」；改为 **4 颗金星**（未满则灰星或空心星）。
- **👁 挪到行尾**：顺序定为 `▾ | 标题区 | [单元四星] | 👁`。
- 箭头 **只负责折叠**，不与星星抢「完成」语义。

### 2.3 折叠态 · 标题排版（日语优先 · 两行）

用户要求：这是日语学习 App，**日语大、中文小、两行**；折叠后视觉中心落在「第 N 单元」这一行日语上。

建议结构：

```text
▾   第1单元              ★★★★  👁
    小李赴日（小字灰，第二行，左对齐与上行对齐）
```

| 元素 | 样式建议 |
|------|----------|
| 第 N 单元 | 15–16px / 800 / `#263238` |
| 场景中文 | 11px / 400 / `#90a4ae`，`line-height` 1.3 |
| 对齐 | 左对齐；**不**用单行「日+中」横挤 |
| 竖直居中 | 整块标题区相对行高垂直居中；箭头与标题区中线对齐 |

与 `PRODUCT-PM` §4「中日双语」一致：**先日文中场景**，不是「中文主标题」。

### 2.4 展开态

- 标题区同上；下方课表 **不再重复** 单元级四星（避免双份）。
- 课次行保留：**课号 + 日标题 + 四关星 + 课图**；点行进课。

## 3. 「4/4 课」有没有意义？

| 角度 | 说明 |
|------|------|
| **对学员** | 像后台计数，不如 **四颗星** 直观 |
| **对家长** | 「4/4」需解释；星星 = 四课全过更好懂 |
| **技术** | `curriculumUnitProgress` 仍可在 `title` 或无障 aria 里读屏 |

**建议（推荐 A）**：

- **默认不显示** `4/4 课`。
- 单元折叠行用 **4 颗单元星**（每课 1 星亮 = 该课四关满；或每星代表一课，四星全亮 = 单元完成 — 见下条需拍板）。
- 长按/无障碍：`aria-label="本单元 3/4 课已通关"`。

**单元四星两种映射（二选一）**：

| 方案 | 规则 | 优点 |
|------|------|------|
| **U-A** | 1 星 = 1 课四关满 | 与「四课」一一对应，好理解 |
| **U-B** | 4 星 = 四关维度聚合（少见） | 与课内四关同名，易混 |

**推荐 U-A**：单元行 4 星 = 第 1–4 课是否「课次通関」。

## 4. 星星 · 眼睛 · 条带图 — 同一套「通关」故事

### 4.1 语义合并

```text
课次四星满 → 课图变彩 + L1 彩蛋可开
     ↓ 本单元 4 课均满
单元四星满 → 行尾 👁 可点 → 弹出 L2 单元四格条带（全彩）
     ↓ 全册 24 课均满
L3 二十四景 / 全家福（灰→彩）
```

- **星星** = 进度与门槛（软件内唯一「完成度」主符号）。
- **👁** = 仅「看画」（奖励层），不表示「未完成」。
- 未通关：**不显示 👁**（与现逻辑一致）；条带在收藏里为灰或锁定剪影。

### 4.2 「五个五角星」？

- 四课 → **4 星** 足够；第 5 星易误解为「满分之上」。
- 若坚持「一眼满」，可用 **第 4 星略大/金色** 或 **四星 + 小👁**（已 proposed），不必 5 星。

### 4.3 彩色图 vs 灰色图（技术建议）

| 做法 | 实现 | 版面 | 推荐 |
|------|------|------|------|
| **CSS 灰度** | `filter: grayscale(1) opacity(.55)` 同一张 webp | 最简；与星状态同源 | ✅ 首选 |
| **双资产** | `strip-color.webp` / `strip-gray.webp` | 体积×2、易不同步 | 仅印刷级需要时 |
| **剪影 + 问号** | 未解锁占位图 | 适合 L1 课图 | 课级保留 |

**规则**：`unitComplete` → 条带/课图 **去灰度**；否则灰。与星星 **同一 `state` 驱动**，避免两套逻辑打架。

## 6. 「我的」页 · 目标信息架构（V1 / V2）

### 6.1 V1（可先落地，约 1–1.5 天）

1. **顶栏两格（或三格）**  
   - 累计 **学习天数**（`studyDays.length`）  
   - 累计 **学习时长**（需 `dailyLog` 累加分钟；无则先显示「—」或仅今日估时）  
   - 今日是否打卡 ✓  

2. **六单元折叠**（与首页 **不同组件**）  
   - 折叠条：**仅** 单元名（日+中两行）+ **4 星摘要** + **细进度条**（4 段，每段一课）  
   - **不** 放地图、不放大图 👁（👁 留在课文页）  
   - 展开：4 行课档案 — 四星 + 状态「通関/進行中」+ 点行进课（「继续学」入口）  

3. **与首页差异**  
   - 首页：进课为主；我的：看档案为主  
   - 文案禁用「完了」；统一 **通関** + 星  

### 6.2 V2（笔记 · 需单独立项）

| 层级 | 入口 | 内容 | 存储草案 |
|------|------|------|----------|
| **课笔记** | 课内底部/侧栏「📝」 | 纯文本 MVP；手写为 P2 | `state.notes.lesson[id]` |
| **单元笔记** | 单元折叠条右侧 📒 | 汇总本单元 4 课笔记摘要或独立字段 | `state.notes.unit[id]` |
| **全册笔记本** | 我的顶部「笔记本」 | 树：全册 → 单元 → 课 | 同一对象只读聚合 |

**折叠态单元右侧显示「有笔记」**：角标 `📝` 或圆点，不展开条带全文。

**技术顺序**：课笔记 → 单元摘要 → 全册页；手写 Canvas **不纳入 V1**。

### 6.3 仍不做在「我的」里的

- 与首页同构的地图 + 六单元课表克隆  
- 开发验收四格按钮（仅 `testcard` + 桌面外壳）

## 7. 六单元配色 · 可读性统筹

### 7.1 现状痛点（用户反馈）

- 部分 **灰字过浅**（`#90a4ae` 在浅粉/浅紫底上对比不足）  
- 部分 **标签过艳**（Spectrum 色块 + 粗字抢眼）  
- 六单元 **左边框色** 与 **标题/进度** 未形成统一层级  

### 7.2 建议（设计 tokens · 待美工确认）

对每一单元固定三色角色：

| 角色 | 用途 | 约束 |
|------|------|------|
| **Accent** | 左边框、当前单元描边 | 饱和度中等 |
| **Surface** | 折叠行背景 | 白或 `#fff` + 5% accent |
| **Text primary** | 日语标题 | ≥ 4.5:1 对 Surface |
| **Text secondary** | 中文副标题 | ≥ 4.5:1（建议 `#607d8b` 而非 `#90a4ae` 在浅底） |

**动作**：从 `CURRICULUM_STAGE_THEMES` 导出一张 **六色对照表**（字色/边框/徽章），在 `封面配色说明.md` 与 `mvp.css` 同步改一版；**禁止**每页临时写色。
