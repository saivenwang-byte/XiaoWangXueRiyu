## 0. 用户拍板（2026-05-25 · 记入母库）

| # | 定案 | 神经/视觉理由 |
|---|------|----------------|
| 1 | **折叠态单词行尽量窄、尽量密** | 一屏多词 → 降低滚动焦虑；信息横向铺满，优先 **一行**，必要时 **两行** |
| 2 | **每词白底 + 细框** | 边界清晰，扫读时「一格一词」 |
| 3 | **注意词行内可淡黄**；**延伸展开区浅灰 `#f5f5f5`** | 黄只表示「警示」；灰表示「延伸阅读」，避免黄/黄相近误判 |
| 4 | **听录播统一灰、无底框**；**折叠彩三角**；**注意黄路标** | 功能色分工，减少「这是什么按钮」的认知切换 |
| 5 | **U1L1 为样品课** | 未用户要求不批量改 2～24 课 |

## 2. 单词关 · 高密度布局规范（折叠态）

### 2.1 一行优先（字段顺序）

```
[序号] [▸?] [日文] [かな·语调·词性] [注意|延伸|活用] [中文义…] ——→ [🔊][🎤][▶]
```

- **▸**：仅有延伸/活用时显示；无则 **22px 占位**，列对齐。
- **中文义**：与核心横排；极窄屏（≤360px）可 **独占第二行**。
- **听录播**：始终贴右，`margin-left: auto`；触控区 **26px**（略小于全局 28px 以省宽）。

### 2.2 两行允许条件

| 触发 | 第二行内容 |
|------|------------|
| 中文义过长 | `l1-vocab-zh-inline` 折行 |
| 窄屏 | 同上（媒体查询） |
| 展开延伸/活用 | 灰底区 `l1-vocab-extra`（不计入折叠态行高预算） |

### 2.3 色分工（防误读）

| 区域 | 背景 | 含义 |
|------|------|------|
| 普通词卡 | 白 + 灰框 | 标准词 |
| 注意词卡 | 淡黄 + 黄左边框 | 需警惕，未展开也要看见 |
| 延伸展开 | **浅灰** | 补充阅读，不是警示 |
| 页底 supp | 白/灰列表 | 全课级说明，不是一词级 |

## 3. 视知觉检查清单（改 UI 前自问）

1. **同时出现的黄有几种含义？** 应 ≤2：行底「注意」+ 路标图标；**不要**在延伸区再用黄底。
2. **灰用于什么？** 仅 **声音相关** 与 **次要 meta 字**；折叠引导 **不用灰三角**。
3. **扫一眼能否数清一行几个词？** 框线 + 4px 间距 + 低 padding。
4. **拇指区**：三键在右缘，是否误触折叠▸？▸ 与三键间距 ≥4px。
5. **WCAG 近似**：黄底 `#fff8e1` 上橙字 `#e65100` 用于小字角标尚可；正文日文勿用浅黄字。

## 5. 会议纪要摘要（可引用）

**议题**：U1L1 单词高密度 + 全课 UI 统筹  
**结论**：

1. 折叠态以 **横向铺满** 为默认，**白盒细框** 划界。  
2. **黄=注意，灰=延伸**，禁止黄延伸底。  
3. 图标 **三类分工**（灰声音 / 彩折叠 / 黄警示）写入 L3 专册。  
4. 本文档为 **以后同类设计的前置条件**；推广它课前须复跑 §1 表。
