# 第1課 · 话筒（录音）功能排查说明

> 2026-05-25 · 与用户反馈「话筒好像没有录音」对照

---

## 1. 结论（先说）

| 区域 | 有无话筒按钮 | 是否应能录音 | 绑定 |
|------|--------------|--------------|------|
| **単語** 每词右侧 | ✅ 有 | ✅ 应能录 | `ShadowSpeak.bind`（`lesson-1-flow.js`） |
| **会話** B 的 A/B/C 回答行 | ✅ 有 | ✅ 应能录 | `DialogueGate.bindSpeak` → 展开场景时 `bindSpeak(body)` |
| **会話** A 开场 / NPC 反应 / 分支选项 | ❌ 隐藏 | 仅 🔊 听 | 故意 `dg-actions-col--listen` |
| **文法** 例句 | ❌ 仅喇叭 | 无跟读需求 | 仅 `SpeakUI` |
| **作業 / 拡張** | ❌ 无话筒 | 文本作业 | — |

**代码链路正常**：所有带 `data-ss-record` 的按钮由 `js/shadow-speak.js` 统一处理；第1課单词与会話 B 行均已生成该属性。

若「点了没反应」，常见为 **环境** 而非按钮未绑定（见 §3）。

---

## 2. 使用方式（避免误以为坏了）

1. **点一次 🎤**：申请麦克风 → 允许后开始录（按钮变红）。
2. **再点一次 🎤**：结束录音；随后 **▶** 可回放（未录前 ▶ 为灰）。
3. **会話 B 行**：录完后点 **✓** 才出评分；A 行无话筒。
4. **静音 3 秒** 会自动结束（单词/会話轻量模式）。

---

## 3. 环境限制（已加强提示文案）

| 环境 | 现象 | 处理 |
|------|------|------|
| `file://` 双击 html | 无法稳定录音 | 已 toast：请用「打开本地预览.bat」→ `localhost:8765` |
| **微信内置浏览器** | 常无法 `getUserMedia` | 已改提示：··· → **在浏览器中打开**（旧文案误写「请用微信打开」已修正） |
| 麦克风拒绝 | 无录 | 地址栏 🔒 → 允许麦克风后重试 |
| GitHub Pages https | 一般可用 | 需用户授权 |

---
