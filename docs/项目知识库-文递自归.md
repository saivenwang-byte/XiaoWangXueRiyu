# 项目知识库 · 文递自归

> **概念定名**  
> - **中文标准名**：文递自归  
> - **技术标准名**：自回归生成 · **Autoregressive Generation**  
> - **在本项目中的用途**：描述 Agent / 开发者**如何迭代改代码**——每一轮新改动必须严格依托「此前已全部确认的内容」递推，禁止每会话从零重写（与 LLM 文本生成机制同构，见 §八）。

## 一、核心定义

文本生成时，模型**逐个输出**最小语义单元 **token**；每一个新内容都**严格依托此前全部已生成内容**作为前置条件；生成内容**实时回流**并入上下文，**循环迭代**输出，直至内容完结。

---

## 二、运行逻辑

| 步骤 | 说明 |
|------|------|
| 1 | **初始**：以用户输入提示词作为首轮基准输入 |
| 2 | **单步**：单次仅生成**单个 token**，不批量整段产出 |
| 3 | **回流**：新生成内容立刻合并进历史上下文，作为下一轮预测依据 |
| 4 | **闭环**：持续往复推导，形成闭环递推输出，直至结束 |

---

## 三、技术等价术语

- 自回归解码（Autoregressive Decoding）
- Token 级自动续写
- 上下文闭环反馈生成

---

## 四、架构归属

**纯解码器（Decoder-only）** 大模型标配生成机制。GPT、LLaMA、文心、通义等主流对话模型均采用该逻辑。

---

## 五、核心特性

| 特性 | 说明 |
|------|------|
| 时序单向 | 仅从前往后生成，**无法逆向修改前文**（已发布/已确认基线同理：回滚须显式指令） |
| 上下文绑定 | 长篇逻辑、语句连贯性稳定——**前文质量决定后文** |
| 分步串行 | 无法并行产出整段，生成速率偏慢 |
| 误差累积 | 局部偏差会向后延续放大（代码里一处递归错误 → 四关全挂） |

---

## 六、区分易混淆概念

| 概念 | 关系 |
|------|------|
| **自编码** | 仅做内容重构，无递推续写 → **与本概念无关** |
| **非自回归生成** | 一次性输出完整内容，无历史回流依赖 → **与本机制相反** |
| **循环神经网络 RNN** | 早期时序模型，原理相近但架构 **不等同** |

---

## 七、能力优缺点

**优势**

- 对话连贯不掉线  
- 行文语法通顺  
- 长内容逻辑自洽  

**劣势**

- 算力消耗更高  
- 生成耗时更长  
- 细微错误易逐步放大  

---

## 八、通俗概括

**文递自归** = 模型**边写边回看全文**，依据过往所有内容接续创作；全程内容**自我回流参照**，一步步完成完整文本产出。

---

## 九、在本项目中的 Agent 映射（文递自归 ↔ 工程迭代）

本仓库**不用**大模型做学员端运行时生成；但 **Agent 改代码** 必须遵守与「文递自归」同构的纪律：

| 文递自归（LLM） | 本项目 Agent 迭代 |
|-----------------|-------------------|
| 用户提示词 = 首轮输入 | 用户**本轮原话** = 唯一新增 scope |
| 已生成 token 序列 | `iteration-baseline.json` → **`confirmed[]`**（已验收，不可静默覆盖） |
| 下一个 token | **本轮最小 diff**（ICE 档位 L1～L4） |
| 合并进上下文 | 冒烟通过 → **写入 `confirmed`**，更新 `current.cache` |
| 非自回归「一次出整段」 | ❌ 新会话**不重写**整站、不批量「顺便优化」 |
| 误差累积 | ❌ 未读基线就改 `speech-engine` → 四关 `読み込みエラー` |

**强制流程**：[Agent文递自归.md](./Agent文递自归.md) 的 **CONFIRM**（C→O→N→F→I→R）  
**禁止**：跳过 `confirmed` 从头设计 UI（等同非自回归、无历史依赖）。
