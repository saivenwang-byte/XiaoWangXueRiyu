# 标日课程生产范式 · MVP 复制模板

> **用途**：第一单元第一课为生产模板，复制到后续所有课次  
> **版本**：v1.0 · 2026-05-25  
> **模板课**：第1課「李さんは中国人です」

---

## 一、思考逻辑（为什么这样做）

### 1.1 核心原则

```
学习者视角 → 课堂真实 → 三个维度
```

| 原则 | 说明 |
|------|------|
| **学习者视角** | 每个功能从「学习时怎么用」出发，不是从「开发怎么方便」出发 |
| **课堂真实** | 模拟真实课堂的信息密度和顺序：单词→会话→语法→练习→总结 |
| **三个维度** | 课本教什么 + 老师补充什么 + 学霸总结什么 |

### 1.2 信息密度控制

```
手机一屏 = 一个主要信息块
展开/折叠 = 隐藏次要信息
标签 + 颜色 = 快速分类识别
```

- 每屏不超过一个主判断
- 次要信息用折叠隐藏
- 颜色只用于分类，不用于装饰

## 二、生产流程（怎么做）

### 2.1 数据生成

```
【产品PRD】源文件 → generate-all-lessons.py → lessons-data.js
```

**生成器位置**：`scripts/generate-all-lessons.py`

**运行命令**：
```bash
python scripts/generate-all-lessons.py
```

**输出**：`js/data/lessons-data.js`（834KB，24课完整数据）

### 2.2 PRD 源文件格式要求

每课 PRD 文件必须包含以下章节（以第1课为模板）：

| 章节 | 标记 | 用途 |
|------|------|------|
| 单词 | `【单词】` | 28词表（仮名/漢字/声調/品詞/中国語） |
| 发音要点 | `発音ポイント` | 4条发音注意事项 |
| 词源注释 | `語源メモ` | 3条词源说明 |
| 活用预告 | `活用予告` | 1条后续课程衔接 |
| 语法 | `【语法】` | 4个ノード（句型/意义/例句/注意） |
| 助词卡 | `【助詞カード】` | 1张助词用法表 |
| 敬语级别 | `【敬語レベル表示】` | 5级敬语对照 |
| 会话 | `【会话】` | 基本课文4句 + 应用课文对话 |
| 会话要点 | `会話のキーポイント` | 5条要点 |
| 角色扮演 | `ロールプレイ課題` | 3个课题 |
| 作业 | `【作业】` | 9节（发音/活用/选择/填空/翻译/改错/作文/听力） |
| 复习扩展 | `【复习与扩展】` | 迄今总结 + 参考表 + 单元测试预告 |

### 2.3 代码渲染

**第1课特有三文件**（其他课暂不复制）：
- `js/lesson-1-flow.js` — L1专用五关流程
- `js/data/l1-knowledge-tips.js` — L1知识点卡片
- `js/data/l1-dialogue-abc.js` — L1会话变体

**通用渲染文件**（所有课共用）：
- `js/app.js` — 主路由（L1单词直接渲染在 app.js 内）
- `js/vocab-flash.js` — 单词关（非L1课使用）
- `js/grammar-network.js` — 文法关
- `js/dialogue-gate.js` — 会话关
- `js/quiz-gate.js` — 作業关

## 三、各模块生产标准

### 3.1 单词关

#### 列布局（左→右）

```
# │ 単語 │ 読み方 │ 調 │ 詞性 │ 意味 │ 🔊🎤▶
```

| 列 | 字段 | 字宽 | 格式 |
|----|------|------|------|
| # | 序号 | 18px | 11px 灰色 |
| 単語 | `v.jp` | 50-62px | 14px bold #333 |
| 読み方 | `v.kana` | 58-68px | 11px #78909c |
| 調 | `v.pitch` | 18px | 12px #b0bec5 |
| 詞性 | `v.pos` | 30px | 9px 橙底白字 |
| 意味 | `v.meaningZh` | flex | 11px #607d8b |
| 操作 | 🔊🎤▶ | 60px | 蓝/红/绿 |

#### 操作按钮

| 按钮 | 颜色 | 功能 | 实现 |
|------|------|------|------|
| 🔊 | 蓝 #1565c0 | 朗读（TTS标准发音） | `SpeechEngine.speakJa(kana)` |
| 🎤 | 红 #d32f2f | 录音（使用者跟读） | ShadowSpeak |
| ▶ | 绿 #2e7d32 | 播放录音（回放） | `SpeechEngine.speakJa(kana)` |

#### 词变标识

- 有变形的词在操作栏显示橙色 `📐` 按钮
- 点击弹出知识卡片，显示活用表
- L1无动词，📐不出现（代码已预留）

#### 发音/语源/预告/敬语区块

位于单词列表下方，使用折叠 `<details>`：
- 🗣 発音ポイント（默认展开）
- 📜 語源メモ
- 🔮 活用予告
- 🎩 敬語レベル表示

### 3.2 会话关

- 场景头：🛬 + 场景名
- 开场白（A）：灰色底单独一行
- B回应：折叠横条 `▶ B：预览文字…`
- 展开后：完整日文 + 中文 + 🔊
- 会话要点：顶部折叠

### 3.3 文法关

- 每节点一卡片：标题行（日+中）+ 日文说明 + 中文说明 + 例句 + 🔊
- `exampleZh` 兼容数组和字符串格式

### 3.4 作業关

- 选择题 / 填空题
- 每题题干 🔊 + 选项 🔊
- 回答后弹出知识卡片

### 3.5 拡張关

- 分类折叠：发音/语法/活用预告/敬语/语源/会话/角色扮演/复习
- 每类内含「先生のひとこと」

## 五、复制到第N课的操作步骤

### Step 1：准备 PRD 源文件

在 `【产品PRD】/新增补课文内容/第N单元/` 下创建 `第N课.txt`，格式参照第1课。

### Step 2：重新生成数据

```bash
python scripts/generate-all-lessons.py
```

验证：`python scripts/_mvp-check.py`，确保 词/法/测/话/作业 全部 ✅。

### Step 3：更新 app.js

如果是第1课拓展到其他课，将 app.js 中 `if (activeGate === 0 && l1)` 的条件改为目标课号。

### Step 4：验证

1. `node --check js/app.js` → 语法通过
2. 浏览器 `v=最新版本号` → 硬刷新
3. 五关逐关验收

## 七、禁止事项

- ❌ 不要在 CSS 中写硬编码颜色（用 CI/VI tokens）
- ❌ 不要修改 `lessons-data.js`（它是自动生成的）
- ❌ 不要在单一文件中做全课批量替换（逐课验证）
- ❌ 不要让手机屏出现横向滚动条
- ❌ 不要在一个折叠里嵌套超过 2 层
