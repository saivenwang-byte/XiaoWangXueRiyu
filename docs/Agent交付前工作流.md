## 一、原则层（骨架 · 5 条）

| # | 原则 | 含义 |
|---|------|------|
| P1 | **先证据后交付** | 未跑自检、未看到 `[OK]` / 通过数量，不得说「已修好 / 可发布」。 |
| P2 | **编号对账，不手查** | 语音 = 喇叭 `data-tts-key` ↔ `tts-cache/{key}.mp3`，用脚本批量比对，不靠肉眼。 |
| P3 | **本地 ≠ 公网** | 本机只用 `http://localhost:8765/index.html?v=当前`；公网只用 GitHub Pages 链接；禁止混用。 |
| P4 | **标日日文独立** | 界面日文按《标准日本语》与 `项目知识库-标日日文书写.md`，中文只进 `*Zh` / `zh` 字段。 |
| P5 | **一次性交互** | 会話评语只服务「当前这一句」；新录音 / 播放 / 换句 → 旧评语必须消失。 |

## 二、项目级硬性要求【反复 ≥5 次】

### A. 语音包（TTS）

| 要求 | 做法 |
|------|------|
| 每条需朗读日文都有 MP3 | `python scripts/audit-tts-registry.py --write`，**缺失 = 0** |
| 喇叭与文件同号 | `data-tts-key` = 文件名（与 `speech-engine.js` `ttsCacheKey` 一致） |
| 清单可核对 | `docs/tts-registry.json`（seq ↔ key ↔ line） |
| 新增文案后 | 跑 `生成语音包.bat` → 再对账 |
| push 必须含 mp3 | 本地有、远程无 = 用户「朗读失败」 |

详见：[TTS-语音包编号规范.md](./TTS-语音包编号规范.md)

### B. 发布与版本

| 要求 | 做法 |
|------|------|
| 发链接前跑全套自检 | `发布前自检.bat` 或 `python scripts/pre-ship-check.py` |
| `?v=` 与 `CACHE_VER` 一致 | `index.html` 全部 script/css 与 `js/share-wechat.js` |
| 交付写清两个链接 | **本地**：`http://localhost:8765/index.html?v=N`；**公网**：`https://saivenwang-byte.github.io/XiaoWangXueRiyu/index.html?v=N` |
| **开发者必须满配** | 链接带 `testcard=1&developer=1`；电脑用 `dev-phone-preview.html`（见 [开发者验收-SOP.md](./开发者验收-SOP.md)） |
| `index.html` UTF-8 无损 | 禁止出现大面积 `??` 或 `</a>` 标签断裂；发版前打开开发者竖屏目检 |
| 未要求不 push | 用户没说 commit/push 时，只改本地 + 给链接说明 |

### C. 本地预览

| 要求 | 做法 |
|------|------|
| 禁止 `file://` 当交付方案 | 双击 html 会空白 / 录音失败；应用 `打开本地预览.bat` |
| bat 自动带当前 v | 从 `share-wechat.js` 读 `CACHE_VER`，打开 `localhost:8765/...?v=` |
| `index.html` 含 file→localhost 跳转 | 误双击 html 时自动跳到 8765 |
| 8765 打不开 / ERR_EMPTY_RESPONSE | 先 `重启本地服务.bat`，再 `打开本地预览.bat`；禁止未起服务就发本地链接 |
| 产品版 vs cache | 产品 **1.0.x** 见 `version-history.json`；`?v=40` 是缓存号，不是 1.40 |

### C2. 双界面 / 双通道验收【项目级·反复 · 2026-05-25】

> **交付真源**：[学员端双界面显示标准-首要执行.md](./学员端双界面显示标准-首要执行.md)（① Cursor 390×844 锁定壳 · ② 微信 https）  
> 开发真源：[双通道验收-浏览器与手机真机框.md](./双通道验收-浏览器与手机真机框.md) · 记忆库 §15  
> **刷新前必读**：[学员端显示界面要求-会议纪要汇编.md](./学员端显示界面要求-会议纪要汇编.md) **§0 清单（R1–R10）** — Cursor 浏览器 / 真机框硬刷新前重读

| 阶段 | 要求 |
|------|------|
| **改码前（前置）** | 双击 **`打开双通道预览.bat`**（= 通道 A 浏览器 + 通道 B 390×844 真机框） |
| **改码中** | 保持 A、B 标签不关；每轮保存后 **两通道均强刷** 再往下改 |
| **交付前（终检）** | `发布前自检.bat` 全 OK 后：**①** 真机壳内目视（切换页壳尺寸不变）+ **②** 微信链 `v=` 一致；开发期 A 全宽可对照；交付块写「双界面已目视」 |
| **通道 B 含义** | `cursor-miniapp-phone.html` 内 iframe = 小程序 WebView 真实使用态，不是桌面全宽代替 |
| **满配四关/彩蛋** | 仍用 `开发者竖屏验收.bat` → `dev-phone-preview.html`，**不替代** B |

### D. ② 会話 UI 与评语

| 要求 | 做法 |
|------|------|
| 🔊🎤▶✓ 右列对齐 | 统一 `dg-line-row` + `dg-actions-col`（NPC 仅 🔊 占位对齐） |
| 页面紧凑 | 少占竖屏；评语在日文行**下方全宽**，不挤在窄列 |
| 评估是第 4 按钮 | 紫色 ✓；**录完不自动评分**，须点 ✓ |
| 评分从严 | 四维各 0–10；无识别不得高分；及格线 70 |
| 评语生命周期 | 全局仅一条；新 🎤 / ▶ / 🔊 / 另一句 ✓ → 清空全部（含按钮旁旧 DOM） |
| 验收 UI | 评语在日文**下方**显示「本句发音」+ 四维 0–10，不是右侧旧版「发音 XX分」 |

### E. 内容与文案

| 要求 | 做法 |
|------|------|
| 单词表 = 标日生词 | `lesson-vocab-biaori.js`，语法不进单词表 |
| 中文灰字 | `meaningZh`、`explanationZh`、`text.zh` 等 + `.zh-annotation` |
| 日文书写 | `python scripts/audit-ja-text.py` |
| 填空朗读 | 用 `questionTts` 完整句，不用带 `＿` 的题干直接 TTS |

## 三、细节层（执行清单）

### 3.1 自动检查（机器）

```bat
发布前自检.bat
```

| 步骤 | 脚本 | 通过标准 |
|------|------|----------|
| 0 | `pre-ship-check.py` → 文递自归基线 | `项目知识库-文递自归.md` + `Agent文递自归.md` 存在；`iteration-baseline` / `version-history` 的 cache = `CACHE_VER` |
| 1 | `pre-ship-check.py` → 版本同步 | index `?v=` = `CACHE_VER` |
| 2 | 单词 meaningZh | zh ≥ 95% jp |
| 3 | 课外讲题 zh | text/list 含 zh |
| 4 | `audit-ja-text.py` | exit 0 |
| 5 | `audit-tts-registry.py --write` | 缺失 MP3 = 0 |

单独语音：`批量检查语音包.bat`

### 3.2 人工快速冒烟（改 UI / 会話 / 语音后必做）

**前置**：`打开双通道预览.bat` → 通道 A `index.html?v=当前` + 通道 B `cursor-miniapp-phone.html`（390×844）均已刷新。

在 **本地** `http://localhost:8765/index.html?v=当前`（并在真机框内复测同一页面）：

1. **第14课 → ② 会話**：三行按钮右对齐；点 🔊 能播；🎤 录完点 ✓ 出评语。
2. **评语切换**：① 评估后 → ② 点 🎤 → ① 评语消失。
3. **第14课 → ② 文法**：文型名 🔊 不「朗读失败」。
4. **单词**：灰字在日文下；🔊 正常。

### 3.3 比对方法（如何知道「对了」）

| 维度 | 比对方式 | 结果记录 |
|------|----------|----------|
| 语音条数 | 脚本输出「需朗读 N · 缺失 0」 | 写入回复 |
| 版本 | `findstr CACHE_VER` vs `index.html ?v=` | 写明 v=N |
| 本地可开 | `python scripts/start-local-server.py --probe` 为 OK | 给出本地 URL；探测失败写「须重启本地服务」 |
| 公网可开 | 仅 push 后；说明「需 push 后公网才更新」 | 给出公网 URL |
| 会話评语 | 冒烟步骤 2 通过 | 写明「已自测切换清除」 |

## 四、交付前循环（RDCA · Agent 必执行）

```
┌─────────────┐
│ R 准备      │ 读 AGENTS.md + 本文件 + 相关 docs；确认本次改动范围
└──────┬──────┘
       ▼
┌─────────────┐
│ D 执行      │ 改代码 / 数据 / 语音包；递增 CACHE_VER
└──────┬──────┘
       ▼
┌─────────────┐
│ C 检查      │ 跑 发布前自检.bat；失败则修复，不进入下一步
└──────┬──────┘
       ▼
┌─────────────┐
│ A 比对反馈  │ 按第三节表格填「交付反馈块」（见下）
└──────┬──────┘
       ▼
┌─────────────┐
│ 交付用户    │ 仅当 C 全 OK + 冒烟无阻塞
└─────────────┘
```

**禁止**：自检未跑就发最终链接；只给公网链接让用户在本机打开；口头说「应该好了」。
