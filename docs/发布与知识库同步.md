# 发布与知识库同步 · 操作规划（MVP v26+）

> 适用：第 14 / 16 / 18 课 H5；后续扩展其它课次仍用本流程。

---

## 一、先结论：选哪种「可发送版本」？

| 方案 | 是什么 | 建议 |
|------|--------|------|
| **A. 公网 H5（推荐）** | 根目录 `index.html` 部署到 **Netlify**，得到 `https://xxx.netlify.app`，微信里发**链接** | ✅ **唯一对外发行版** |
| B. GitHub Pages | `saivenwang-byte.github.io/XiaoWangXueRiyu/...` | 仅作备份/电脑浏览器；**国内微信常打不开** |
| C. 「手机微信版」子文件夹 | 旧入口，代码已指向 `../js` 同一套 | ❌ 不要再单独维护一份 |
| D. 「发布包」文件夹 | 历史打包副本 | ❌ 仅当 Netlify 拖错目录时的备用；**以根目录为准** |
| E. 微信小程序 | `japanese_learning_miniapp/` 骨架 | 未来另立项；**与当前 MVP 无关** |

**推荐做法：只有一套代码、一种分享方式**

- **开发真源**：本机 `日语学习` 根目录（含 `tts-cache/`）。
- **用户看到的「微信版」**：= 同一个 H5 的 **HTTPS 链接**（不是再做一个 app）。
- **知识库（Notion / 飞书 / 豆包文档等）**：存**链接 + 版本号 + 变更说明 + 验收清单**，**不要**在知识库里再复制一整份 HTML/JS。

## 二、本地 / 知识库 / 线上 各自放什么？

```
┌─────────────────────────────────────────────────────────┐
│ 本地 D:\...\日语学习          ← 唯一开发源（Git main）   │
│   index.html  js/  css/  tts-cache/  docs/            │
└───────────────────────────┬─────────────────────────────┘
                            │ git push 或 Netlify 拖拽
                            ▼
┌─────────────────────────────────────────────────────────┐
│ 线上 Netlify（主） + GitHub Pages（备）                  │
│   https://你的域名.netlify.app/index.html?v=26         │
└───────────────────────────┬─────────────────────────────┘
                            │ 只同步「元信息」
                            ▼
┌─────────────────────────────────────────────────────────┐
│ 知识库 1 页「标日课后巩固 · 发行版」                    │
│   · 当前正式链接（Netlify）                             │
│   · CACHE_VER / 日期 / 课次范围                         │
│   · 发给学员的复制话术                                  │
│   · 验收打勾表（见第四节）                              │
│   · 扩展新课时的数据文件清单                            │
└─────────────────────────────────────────────────────────┘
```

**更替规则**

1. 每次发版：先改本地 → 验收 → 部署线上 → **最后**改知识库里的链接与版本（避免学员点到半成品）。
2. 知识库**不要**替代 Git；代码历史以 Git 为准。
3. `VERSION-WECHAT-v1.md` 保留作**历史冻结说明**；当前活跃版本建议新增 `VERSION.md`（或更新 README「当前 build」一行）。

## 三、发版前全局验收（约 20 分钟）

在 **`http://localhost:8765/index.html?v=当前版本`** 完成（禁止 `file://`）。

### 3.1 自动化

```bat
python scripts/verify-tts-cache.py
```

- 有 `[MISS]` 时：先 `python scripts/gen-missing-mp3.py` 或 `生成语音包.bat`，再复检。
- L14 第 5 题应存在 `荷物を持ちましょうか。` 对应 MP3。

### 3.2 三课 × 四关（每课抽 2 分钟）

| 关卡 | 检查点 |
|------|--------|
| 単語 | 每行 🔊🎤▶；连点不同行 🎤 不弹「请先结束录音」；快速连点 🔊 只响最后一句 |
| 会話 | 开场 + ①②③ 各有 🔊🎤▶；中文注释小字 |
| 文法 | 标题/说明 🔊；例句每句旁 🔊；先生の補足 ②③ 🔊 |
| テスト | 上一题/下一题；题干+选项 🔊；连点选项不叠音 |

### 3.3 微信实机（部署后必做）

1. 手机 **4G** 打开 Netlify 链接（关 WiFi 测一次）。
2. 点 🔊：应优先本地 MP3，不应读成中文。
3. 跟读：允许麦克风；不行时按提示「在浏览器中打开」。

## 四、对外发布操作步骤（推荐顺序）

### 步骤 1 · 锁定版本号

- `index.html` 全部 `?v=26`（或递增）。
- `js/share-wechat.js` → `CACHE_VER = "26"`。
- `index.html` 内 Service Worker key → `hyouga-sw-purged-v26`。

### 步骤 2 · 提交 Git（可选但强烈建议）

```bat
git add index.html js css docs tts-cache scripts
git commit -m "release: MVP v26 单通道朗读与 L14/16/18 验收"
git push origin main
```

（若 push 仍 reset，可跳过，直接步骤 3 拖拽部署。）

### 步骤 3 · Netlify 部署（微信主渠道）

1. 双击 `打开Netlify拖拽部署.bat`。
2. 拖入整个 **`日语学习` 根文件夹**（含 `index.html`、`tts-cache/`）。
3. 记下 `https://xxxx.netlify.app`。
4. 编辑 `js/public-url.config.js`：

   ```js
   window.HYOUGA_PUBLIC_ORIGIN = "https://xxxx.netlify.app";
   ```

5. **再拖一次**同一文件夹（让分享按钮复制 Netlify 链接）。

### 步骤 4 · 自检公网

- `https://xxxx.netlify.app/index.html?v=26`
- `https://xxxx.netlify.app/share.html` → 应显示可分享

### 步骤 5 · 同步知识库（1 页即可）

在知识库更新或新建「发行版」条目，字段建议：

| 字段 | 示例 |
|------|------|
| 正式学习链接 | `https://xxxx.netlify.app/index.html?v=26` |
| 分享页 | `.../share.html` |
| 版本 | v26 / 2026-05-21 |
| 课次 | 14 / 16 / 18 |
| 变更摘要 | 单通道朗读、录音切换、测验导航… |
| 学员话术 | 见 `share.html` 或 README |
| 备份链接 | GitHub Pages（注明微信可能打不开） |
| 下次扩展 | 见第五节 |

### 步骤 6 · 微信发送

复制 **Netlify 的 index 链接** 到聊天；**不要**发 zip、不要发 `github.com/仓库` 地址。

## 五、后续扩展其它课（14/16/18 之外）

### 5.1 数据层（只改 JS，不改壳）

| 文件 | 作用 |
|------|------|
| `js/data/lessons-mvp.js` | 课次骨架：单词、会話、文法节点 |
| `js/data/lessons-mvp-depth.js` | 测验 `quizReplace`、文法 `DEPTH_PARSE_HINTS`、选项假名 `QUIZ_OPTION_KANA` |
| `【产品PRD】/*.txt` | 课文与讲解原文（导入参考） |

每加一课建议顺序：

1. 在 PRD/教材中定稿日文 + 中文注释。
2. 写入 `lessons-mvp.js`（复制 14 课结构改 id）。
3. 补 `lessons-mvp-depth.js`（测验、denseList、questionTts）。
4. `python scripts/build-tts-cache.py`（全量）+ `verify-tts-cache.py`。
5. 本地三关 + 测试关验收。
6. `?v=` + `CACHE_VER` +1 → 部署 → 知识库改「课次范围」。

### 5.2 不建议的做法

- 为每课复制一个 `手机微信版` / `发布包` 目录。
- 在知识库贴完整 JS 源码（难以 diff，易过期）。
- 未跑 `verify-tts-cache` 就发微信（易出现「朗读失败」）。

### 5.3 版本策略（简单版）

- **小改**（修 bug、补 MP3）：同一仓库 `main`，`v` 递增（27、28…）。
- **大改**（换 UI、拆仓库）：按 `VERSION-WECHAT-v1.md` 打 Git 标签后新开仓库或 `v2` 分支。
