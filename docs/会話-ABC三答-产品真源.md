## 1. 一场景在讲什么（先分清「问」和「答」）

| 顺序 | 角色 | 内容来源 | 界面表现 |
|------|------|----------|----------|
| **① 对话话题 / 发起句** | 对方（NPC） | **标日应用课文** `dialogues[].opener` | 日文一行 + 其下灰色 **中文释义**（教学用，不受「关中文」误伤时用 `showPedagogyZh`） |
| **② 轮到学员** | B · あなたの番 | — | 短提示：`選ぶ回答 A＝课文 · B/C＝场景变体（均可沟通）` |
| **③ 三种回答** | 学员侧 **A / B / C** | `*-dialogue-abc.js` 每场景 **3 条** `replies` | 见 §2；**必须三条都在** |

**不是对错题**：B/C 不是「错项」，是同一问在不同场合下的**自然说法**；三条均可沟通。

## 2. A / B / C 各自是什么意思（全课统一）

### A ＝ 课文标准答

| 行 | 字段 | 要求 |
|----|------|------|
| **第 1 行** | `japanese` | 标日该句 **应用课文原句**（主线答案） |
| **第 2 行** | `chinese` | 该句 **中文翻译 / 释义**（灰字，在日文**下方**，独占一行） |

- 标签：**A** + 小字 **「课文」**（橙系色条，与课2–24 一致）。  
- **A 以两行正文为主**；若数据有 `noteZh`，仅作「这是课文句」的短说明，**不能**替代第二行翻译，也**不能**省掉 B/C。

### B / C ＝ 场景变体（辩题 / 沟通变体）

| 行 | 字段 | 要求 |
|----|------|------|
| **第 1 行** | `japanese` | 变体日文 |
| **第 2 行** | `chinese` | 变体中文释义（日文下、灰字） |
| **第 3 行** | `noteZh` | **中文**补充：在什么**场景**、对**什么人**说更合适；并点出与 A / 其它变体的**差异**（例：更短 / 更礼貌 / 更正式） |

- 标签：**B** +「变体B」（蓝）、**C** +「变体C」（绿）。  
- **第三行必须直接可见**（普通灰字段落），**不是**只能点「提示 ▾」才看见；也**不是**与第二行挤在同一行。

### 三者关系（给用户的一句话）

```text
同一句课文问话 → A 照课文答 → B/C 是真实生活里换了一种说法，仍然说得通；
第三行中文帮你记住「什么时候用哪句」。
```

## 3. 数据契约（每场景 · 每课）

| 键 | 必填 | 说明 |
|----|------|------|
| `replies` | ✅ 长度 **3** | 顺序建议 A(rank1) · B(rank2) · C(rank3) |
| `label` | ✅ | `"A"` / `"B"` / `"C"` |
| `japanese` | ✅ | 日文句 |
| `chinese` | ✅ | 中文释义（A/B/C 都要有） |
| `noteZh` | B/C **必须有**；A 可选 | 场景与差异说明；见 §2 |
| `abcGuideZh` | 推荐 | 场景顶上的黄/灰 **整句引导**（本句谁答、三答关系） |

## 4. UI 结构（`renderL1AbcReplyList` 应对齐的 DOM 语义）

```text
┌─ 场景展开 ─────────────────────────────┐
│ [发起句] 日文                          │
│          中文（下）                     │
│ B · 選ぶ回答 A＝课文 · B/C＝场景变体    │
│ [可选] abcGuideZh 整段引导              │
├─ A 卡 ───────────────────────────────┤
│ A [课文]  🔊…                          │
│   日文……………………（块级第1行）            │
│   中文……………………（块级第2行）            │
│   （A 若有 noteZh：短说明，非折叠）     │
├─ B 卡 ───────────────────────────────┤
│ B [变体B]                              │
│   日文                                 │
│   中文                                 │
│   场景说明 noteZh（第3行 · 必见）       │
├─ C 卡 ───────────────────────────────┤
│   同上                                 │
│ [npcReaction 若有]                     │
│ [本句听完 ✓]                           │
└──────────────────────────────────────┘
```

**禁止**

- 只显示 A、隐藏 B/C  
- 把 B/C 收成「另一种说法」一行带过  
- 把 `noteZh` **只**放进 `SenseiTipCard` 折叠条（`提示 ▾`）而默认看不见第三行  
- 日文与中文并排同一行（违背「上日下中」）  
- 用 ①②③ 替代 A/B/C 标签（非 MVP 五关 ABC 课）

## 7. 验收口诀（真机 · 任一课会話）

1. 展开任一场景 → 先看到 **课文发起句（日+下中）**。  
2. **A / B / C 三张卡都在**；标签与色条对。  
3. **A**：日文下必有 **整句中文**。  
4. **B、C**：日文 → 中文 → **第三行场景差异中文**（不用点开提示才看见）。  
5. 点 🔊 / 跟读、**本句听完**、场景折叠行为与改前一致。
