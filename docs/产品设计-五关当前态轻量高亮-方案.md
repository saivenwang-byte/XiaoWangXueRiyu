# 产品设计 · 第1課五关「当前版块」轻量高亮方案（美工意见）

> **状态**：**方案 A 已落地**（2026-05-25）  
> **问题来源**：用户反馈 ——「对应版块」逻辑对，但 `3px double` + 上下 inset 线 **过重、显丑**  
> **约束**：[平面轻量UI.md](./产品设计规范-平面轻量UI.md) · [DESIGN-L0](./DESIGN-L0-图标与人设全景.md) · 五关分色已定（蓝/绿/橙/紫/青绿）

---

## 1. 要解决什么（产品语义不变）

| 必须保留 | 说明 |
|----------|------|
| **一眼知「正在哪一关」** | 点「拡張」时，顶栏 + 中间列表 + 底栏传送链，都应是**同一关**的视觉语言 |
| **五关可区分** | 颜色仍按 gate 0/2/1/3/4，不与 L3 听录播灰、注意黄混用 |
| **扁平轻** | 不用双线框、不用厚阴影、不用「相框套相框」 |

| 建议拿掉 | 原因 |
|----------|------|
| `border: 3px double` | 像公章/老式 WinForms，和课内细框单词行不协调 |
| `inset` 上下粗色条 + 外框双线 | 三层描边叠在一起，**信息重复**且占高度 |
| 内容区整圈粗框 | 列表已经很密，外框再抢视线 |

## 2. 当前版为何显丑（美工自检）

```text
┌─ double 外框 ─────────────────┐
│════ 上 inset 色带 ═══════════│  ← 与 Tab 上沿「再画一遍」
│  列表内容…                    │
│════ 下 inset 色带 ═══════════│
└──────────────────────────────┘
```

- **重复**：Tab 已有双线 + 色带，内容区再来一套 → 像「强调强调再强调」  
- **过重**：`3px double` 在 375px 屏上约 6～9px 有效边宽，挤压正文  
- **不巧**：没有「从 Tab 流到内容」的**一条视觉动线**，只是两块同色框并排

---

## 3. 推荐方案（三选一）

### 方案 A · **左侧色条桥接**（推荐 ★）

**想法**：用 **一条 4px 竖色条** 贯穿「当前 Tab ↔ 内容区」，像书签/目录左侧指示条；其余保持白底细线。

```text
顶栏
 [単語] [会話] [■文法] [作業] [拡張]
              ↑
         左 4px 橙条 + 浅橙底（无双线）

内容区
 ┃  1  …折叠列表…          ← 仅左侧 4px 橙条，无整圈框
 ┃  2  …
 ┗━━━━━━━━━━━━━━━━━━━━
     底栏传送链（顶部分隔 1px 浅橙即可）
```

| 元素 | 样式 |
|------|------|
| 当前 Tab | 背景 `#fff8e1`（约 8% 橙 tint）· 边框 `1px #ffe082` · **左边 `4px solid #e65100`** |
| 非当前 Tab | 保持现有浅 pastel + `1px` 细边 |
| 内容 `.l1-gate-panel` | **无整圈框** · 白底 · **仅 `border-left: 4px solid` 与 Tab 同色** |
| 滚动区 | 右侧保留细滑槽，不加外框 |

**优点**：轻、巧、动线清晰；和单词行「左 3px 注意黄条」语言统一。  
**缺点**：极窄屏下 4px 要确认不挡字（内容区已有 padding-left）。

---

### 方案 B · **底边指示条**（更极简）

**想法**：借鉴分段控件 —— 当前 Tab **底边 2px 实色条**；内容区顶边 **1px 同色线** 即可关联，不要框住列表。

```text
 [単語] [会話] [文法] [作業] [拡張]
              ━━━ 2px 橙底条

 ─── 1px 橙线 ─────────────────
  列表…
```

| 元素 | 样式 |
|------|------|
| 当前 Tab | 白/极浅底 · **仅 `border-bottom: 2px solid` 关色** · 无左右粗框 |
| 内容区 | **`border-top: 1px solid` 同色** · 无 left/right/bottom 框 |

**优点**：最省像素、最「轻」。  
**缺点**：Tab 与内容中间若有空隙，关联感略弱于 A（可用同色浅底垫一层弥补）。

---

### 方案 C · **浅底同 tint + 1px 单线**（折中）

**想法**：保留「整块浅色」暗示当前关，但 **只保留 1px 单色实线**（禁止 double / inset）。

| 元素 | 样式 |
|------|------|
| 当前 Tab | 背景 = 关色 12% tint · `1px solid` 关色 · `font-weight: 700` |
| 内容区 | 背景 = 关色 **6%** tint · `1px solid` 关色 · `border-radius: 6px` |

**优点**：比现版轻很多，改动小。  
**缺点**：仍有一圈框，列表密时略闷；不如 A 有「一条线串起来」的巧劲。

## 6. 美工建议结论

| 优先级 | 选择 | 一句话 |
|--------|------|--------|
| **首选** | **方案 A** | 左侧色条桥接 Tab 与内容，轻、巧、符合平面轻量 |
| 备选 | 方案 B | 若要「几乎看不见框」 |
| 可接受 | 方案 C | 若你仍想要「整块浅底」但去掉双线 |

**不建议** 继续用双线 + inset 上下色带。
