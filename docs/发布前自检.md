# 发布前自检

每次要发微信链接或 push 到 GitHub Pages 之前，在本机执行：

```bat
发布前自检.bat
```

全部显示 `[OK]` 后再 `git push`。

## 日文书写（标日对齐）

规范全文：**[项目知识库-标日日文书写.md](./项目知识库-标日日文书写.md)**

```bash
python scripts/audit-ja-text.py
```

常见错误：中文逗号 `，`、半角 ` / ` 并列、`買い物しました` 漏「を」、`kana` 里用 `／` 而非 `・`。

## 语音包为什么会「丢」

1. **新增了日文文案但没生成 MP3**（例如 `lesson-vocab-biaori.js` 加入后未跑 `生成语音包.bat`）
2. **`verify-tts-cache.py` 扫描文件不全**，误以为通过（已要求扫描 `lesson-vocab-biaori.js`）
3. **push 时未提交** `tts-cache/*.mp3`（文件在本地有，远程仓库没有）
4. **微信内**不用 Google 在线 TTS，主要靠 `tts-cache/`；缺 MP3 就只能听系统音或无声

修复：运行 `生成语音包.bat`，再 `发布前自检.bat`，确认无 `[MISS]` 后 push。

## 中文灰字（课外讲题 / 单词 / 语法）

| 位置 | 数据字段 | CSS |
|------|----------|-----|
| 单词 | `meaningZh` | `.zh-annotation` 在日文 `meaningJa` 下方 |
| 语法说明 | `explanationZh` | `.gn-explain-zh.zh-annotation` |
| 语法例句 | `exampleZh` / `GRAMMAR_EXAMPLE_ZH` | `.gn-example-zh.zh-annotation` |
| 课外讲题 | `text.zh` 或 `list[].zh` | `.gn-dense-zh.zh-annotation` |
| 会話 | `noteZh` | `.zh-annotation` |

用户关闭「显示中文」时，`html[data-show-zh="0"]` 会隐藏所有 `.zh-annotation`。
