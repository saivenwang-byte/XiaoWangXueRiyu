
## 双通道目视（机器 OK 后 · 交付前终检）

1. 双击 **`打开双通道预览.bat`**
2. **通道 A**：`index.html?v=CACHE_VER` — 强刷后确认本次改动
3. **通道 B**：`cursor-miniapp-phone.html` — **390×844** 框内确认真实使用态（底栏、折叠、无横滚）

详见 [双通道验收-浏览器与手机真机框.md](./双通道验收-浏览器与手机真机框.md)。

## 文递自归基线（必检 · 第一步）

`pre-ship-check.py` 会校验：

1. `PROJECT_SPEC.md`、`docs/PROJECT_ARCHITECTURE.md`、`.cursor/rules/wendi-zigui-core.mdc` 存在  
2. `docs/项目知识库-文递自归.md`、`docs/Agent文递自归.md` 存在  
3. `docs/iteration-baseline.json` 的 `current.cache` = `js/share-wechat.js` 的 `CACHE_VER` = `index.html` 的 `?v=`  
4. `docs/version-history.json` 的 `current.cache` 与上一致  

Bump 缓存版本后须同步四处（`CACHE_VER` / `index.html` / `iteration-baseline.json` / `version-history.json`），并运行 **`同步作者链接.bat`**，否则自检可能 **FAIL**。

## 语音包编号对账（必检）

每条 🔊 对应编号 `data-tts-key` = 文件 `tts-cache/{key}.mp3`。

```bat
批量检查语音包.bat
```

或 `python scripts/audit-tts-registry.py --write` → 更新 `docs/tts-registry.json`。

| 指标 | 要求 |
|------|------|
| 缺失 MP3 | **0** |
| 需朗读条数 vs 有 MP3 | 每条需求都有文件 |

规范全文：[TTS-语音包编号规范.md](./TTS-语音包编号规范.md)

## 语音包为什么会「丢」

1. **新增了日文文案但没生成 MP3**（文法课外 denseList、单词、questionTts 等）
2. **扫描与页面朗读逻辑不一致**（已用 `tts_lib.py` 对齐 `denseSpeakLine` / `questionTts`）
3. **push 时未提交** `tts-cache/*.mp3`（本地有、GitHub 无 → 浏览器 404）
4. **微信内**主要靠 `tts-cache/`；缺 MP3 会提示「朗读失败」

修复：`生成语音包.bat` → `批量检查语音包.bat` 确认缺失 0 → `发布前自检.bat` → push。

## 中文灰字（课外讲题 / 单词 / 语法）

| 位置 | 数据字段 | CSS |
|------|----------|-----|
| 单词 | `meaningZh` | `.zh-annotation` 在日文 `meaningJa` 下方 |
| 语法说明 | `explanationZh` | `.gn-explain-zh.zh-annotation` |
| 语法例句 | `exampleZh` / `GRAMMAR_EXAMPLE_ZH` | `.gn-example-zh.zh-annotation` |
| 课外讲题 | `text.zh` 或 `list[].zh` | `.gn-dense-zh.zh-annotation` |
| 会話 | `noteZh` | `.zh-annotation` |

用户关闭「显示中文」时，`html[data-show-zh="0"]` 会隐藏所有 `.zh-annotation`。
