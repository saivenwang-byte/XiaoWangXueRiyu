# Agent 文递自归（项目级 · 增量纪律）

> **地位**：与 [Agent交付前工作流.md](./Agent交付前工作流.md) 并列。  
> **交付前工作流** = 发给你之前怎么验（质量闸门）。  
> **文递自归** = 每一轮怎么改（**在上一次已确认基线上叠加**，禁止每会话从零重写导致崩溃）。技术标准名：**Autoregressive Generation**。  
> **概念全文**：[项目知识库-文递自归.md](./项目知识库-文递自归.md)

**机器可读基线**：[iteration-baseline.json](./iteration-baseline.json)（会话开场先读）  
**踩坑与版本**：[Agent交付前工作流-记忆库.md](./Agent交付前工作流-记忆库.md) · [version-history.json](./version-history.json)

## 一、两条铁律（最高优先级）

| # | 铁律 | 违反后果 |
|---|------|----------|
| T1 | **先对齐基线，再动手** | 未读 `iteration-baseline.json` + 记忆库就改代码 → 重复劳动、覆盖已验收功能、引入栈溢出等崩溃 |
| T2 | **用户本轮未要求的，不改** | 擅自改 UI/删模块 → 用户看到「界面消失」；实际是 JS 崩了或 DOM 结构被换 |

---

## 二、迭代刻度 ICE（Incremental Change Escalation）

每轮改动开始前，**自评档位**；只能升档不能跳档（例如未过 L2 验证不得做 L3）。

| 档位 | 名称 | 允许范围 | 必做验证（不可省略） | 须用户本轮明示 |
|------|------|----------|----------------------|----------------|
| **L0** | 只读对齐 | 读基线、解释、排期 | 无代码改动 | — |
| **L1** | 单点修补 | **1 个文件**内 bugfix | 冒烟清单中与该文件相关的 **1 项** | — |
| **L2** | 关内联动 | 同一 Tab/关 2～4 文件 | 该关完整冒烟（见 §五） | — |
| **L3** | 四关 + 语音 | 动 `speech-engine` / 数据 / 多关 UI | `pre-ship-check.py` 全绿 + **四关挂载**（§五 ①～④） | — |
| **L4** | 方案迭代 | 用户已书面同意的 P0～P3 条目 | L3 全部 + 更新 `iteration-baseline.json` 的 `confirmed` | **方案条目**（如「同意全部意见」） |
| **L5** | 禁止擅自 | 删功能、大重构、改未批 UI、换整体信息架构 | — | **逐条明确指令** |

**升档规则**：当前档位验证未写入基线 / 未通过，**不得**在下一轮当作「已完成」跳过。

## 三、每轮 CONFIRM 六步（禁止从头再来）

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ C 对齐  │ → │ O 范围  │ → │ N 收窄  │ → │ F 修复  │ → │ I 入账  │ → │ R 交付  │
│ 基线    │    │ 仅本轮  │    │ 最小diff│    │ +验证   │    │ 基线JSON│    │ 反馈块  │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

| 步 | 动作 | 产出 |
|----|------|------|
| **C** Confirm | 读 `iteration-baseline.json` → `current` + `confirmed[]` + `backlog[]` | 口头/心里明确：**哪些已验收不能再破** |
| **O** Only | 从用户消息提取 **本轮唯一目标**；其余进 `backlog` 不动 | 一句话 scope |
| **N** Narrow | 列出将改文件路径；超过 L2 文件数则拆轮 | 文件清单 ≤ 档位上限 |
| **F** Fix | 改代码；跑档位对应验证 | 命令输出 / 冒烟结果 |
| **I** Inventory | 本轮验收项写入 `iteration-baseline.json`（`confirmed` 或 `backlog` 状态） | JSON 已更新 |
| **R** Release | 仅 L3+：附 [Agent交付前工作流.md](./Agent交付前工作流.md) 第五节「交付反馈块」 | 用户可见结论 |

**禁止**：未做 C 直接 F；未做 F 通过就 I 入账；未做 I 就声称「上一轮已完成」。

## 四、已确认基线 vs 待办（分工）

| 存储 | 文件 | 内容 |
|------|------|------|
| **已确认（勿回归破坏）** | `iteration-baseline.json` → `confirmed[]` | 每条含 `id`、`since_cache`、涉及文件、验收句 |
| **用户同意未做** | 同上 → `backlog[]` | 如 P0 灰字、P2 难词提示；**不得**混进 confirmed |
| **踩坑与费用** | `Agent交付前工作流-记忆库.md` | 根因叙述（如评语 duplicate、localhost 僵尸） |
| **产品/缓存版** | `version-history.json` | semver 与 `?v=` 沿革 |

Agent **不得**用会话记忆代替上述文件；新会话 = 读文件继续叠层（文递自归），不是重写 app。

## 五、回归清单（L3 必跑 · 改 UI/语音/会話后）

在本地 `http://localhost:8765/index.html?v=` **当前 CACHE_VER**（见 `js/share-wechat.js`）：

| # | 检查 | 通过标准 |
|---|------|----------|
| ① | 第14课 **単語** | 有单词列表；**无** `読み込みエラー` |
| ② | 第14课 **文法** | 秘技/文型可见；🔊 不「朗读失败」 |
| ③ | 第14课 **会話** | 三行按钮对齐；评语在日文**下方**；换句旧评语消失 |
| ④ | 第14课 **テスト** | 题目渲染；🔊 读 `questionTts` 非空题干 |
| ⑤ | 机器 | `python scripts/pre-ship-check.py` 无 FAIL |

**已知致命回归**（见记忆库 §7）：`speech-engine.js` 中 `fallbackLines` ↔ `primaryTtsKey` 互调 → 四关全挂。修语音后必跑 ①～④。

## 六、与用户沟通的边界

| 可以说 | 不能说（无证据时） |
|--------|-------------------|
| 「本轮只改 X，不动 Y（基线已确认）」 | 「界面我删了」—— 多数是 JS 崩溃 |
| 「请开 `?v=44` 强刷」 | 「你电脑缓存问题」—— 未确认 CACHE_VER |
| 「P0 灰字在 backlog，你确认四关正常后我做」 | 「顺便帮你优化了整体 UI」 |

用户说 **「不要擅自修改」** → 本轮最高档位 **L1～L2**，且 scope 仅限用户原话。
