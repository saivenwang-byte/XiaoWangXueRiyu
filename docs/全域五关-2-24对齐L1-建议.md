## 四、若要对齐：针对性调整建议（供你拍板）

### 优先级 P0（第1单元 2–4 课 · 阻塞「结构对齐」）

1. **文法**：每 `grammarNodes[]` 补 **`example`（标日原文）+ `exampleZh`（字符串或 string[]）**，禁止空 `[]`。
2. **作業 · 小测**：L2/L3/L4 各 **+1 题** 至 12 题；建议补 **「間違い探し」或综合选择**，`grammarNodeId` 绑当课节点。
3. **拡張**：每课 `reviewExtension` 补至 **5 块**（对齐 L1 标题体系）：
   - 已有：📖 迄今总结、🗂️ 参考表  
   - **待补**：📋 当课句型模板、⚠️ 常见误用（≥4 条 ×/○ 对比）、📝 单元测试预告（第4课写 U1 测范围）

### 优先级 P1（第2单元 5–7 课 · 文法例句）

- 仅补空节点例句，**不动** 会話 ABC / 冻结课 14·16·18 交互壳。
- 真源：`lessons-detailed.js` 同课 → `audit-ja-text.py`。

### 优先级 P2（第9–24 课 · 深度，结构已过）

- 沿用 `docs/MVP-L9-24-严齐L1-补课单.md` 的 P1–P3 批次；**不必回退** 会話为 L1 录音流。

### 优先级 P3（全域 · 单词知识卡）

- L1：`l1-knowledge-tips.js` 逐词。  
- U2：`unit2-knowledge-tips.js` 已有部分 id。  
- 9–24：运行时 `Lessons924KnowledgeTips` **不算未对齐壳**，但若你要 **严齐 L1 体验**，需按课增补 `lN_v_*` 独立文案（工作量大，建议课后批）。

## 五、作業 · 拡張「补全」建议（按 L1 MVP 形式）

### PM 内容真源（2026-05-25）

- **作業**：一律以 **标日课文**（`basicText` + 会話）为出题依据。  
- **拡張**：以标日课文为锚点做 **知识延展**；可略超纲，须标 **【知识延展】**。  
- 全文规范：`docs/作业与拡張-标日真源规范.md`

### 5.1 作業关（`homeworkSections` + `quizQuestions`）

**L1 标准形态**（课1 已落地，可直接当模板）：

| 序号 | 段标题（含关键字） | `lines` 内容形态 | 与 gate3 关系 |
|------|-------------------|------------------|---------------|
| 1 | 発音練習 | 日语词/句 + 中文发音提示 | 独立 |
| 2 | 活用練習 | 动词/形容词变形表或填空 | 独立 |
| 3 | 選択問題 | Q1–Q4 + 选项 A–D + `→ 答案` | 可映射 quiz 1–4 |
| 4 | 穴埋め | Q5–Q7 + `→ 假名/汉字` | 映射 quiz 5–7 |
| 5 | 翻訳（日→中） | 日文句 + `→ 中文` | 映射 quiz 8–9 |
| 6 | 翻訳（中→日） | 中文句 + `→ 日文` | 映射 quiz 10–11 |
| 7 | 間違い探し | 错句 + 错误说明 + 正解 | **quiz 第12题** 或 Q12+Q13 |
| 8 | 作文ミニタスク | 课题 + 要求 + 例文 | 独立 |
| 9 | 聴解（预留） | 题干 + 选项 + `→`（可标 TTS） | 独立 |

**第2–4课现状**：9 段 **已有**，缺的是 **第 12 道 gate 小测** 与部分段内 **Q 编号与 quiz 一一对应**。

**补全动作（建议）**：

```text
每课（2、3、4）：
  1. 在 homeworkSections「間違い探し」增加 Q12（若无）或把现有改错并入第12题逻辑
  2. 在 quizQuestions 追加 lN_q12（type: choice 或 fill，绑 grammarNodeId）
  3. 跑 enrich-lessons-9-24-l1-depth.py 的同类逻辑可为 2–4 写分支，或单脚本 patch-unit1-homework-quiz.py
```

**第5–24课**：结构已 12 题；P2 只做 **文案对标标日练习册**（翻译说明、改错解析），不增段数。

### 5.2 拡張关（`summaryBlocks` + `reviewExtension` + 要点/扮演）

**L1 标准（五块 `reviewExtension`）**：

| 块 | 标题模式 | 内容要求 |
|----|----------|----------|
| 1 | 📖 これまでのまとめ | 本课 1 段总结 + **跨课链接** 2–4 条（第N课：…） |
| 2 | 🗂️ 参考表 | 当课 **こそあど/敬称/数词/动词表** 等渐进表（Tab 分隔可复制） |
| 3 | 📋 …テンプレート | **可替换［］模板** 2–4 句 + 1 条完整例 |
| 4 | ⚠️ よくある誤り | **× / ○** 对比 ≥4 条，对应当课文法 |
| 5 | 📝 ユニットテスト予告 | 单元末测范围、建议复习顺序 |

**`summaryBlocks`（四 key，L2–4 通常已有）**：`pronunciation` / `etymology` / `preview` / `honorific` — 保持与 L1 同 key，只改 `lines` 为当课内容。

**`dialogueKeyPoints` / `rolePlayTasks`**：L2–4 **已有**（6 条要点 + 3 个扮演课题），**无需从 0 补**；9–24 亦有。扩展开关 UI 读 `reviewExtension` 五块为主。

**第2–4课补全草案（每课 +3 块）**：

```json
// 块3 模板 — 示例 L2
{
  "title": "📋 指示詞テンプレート（模板）",
  "lines": [
    "これは［名詞］です。",
    "あれは［人］の［物］ですか。",
    "この［名詞］は［人］のです。",
    "例：この ノートは 私のです。"
  ]
}

// 块4 误用 — 示例 L2
{
  "title": "⚠️ よくある誤り（常见误用）",
  "lines": [
    "× どれですか　○ どれが～ですか",
    "× これ 本　○ この 本",
    "× あれは だれですか（人）　○ あの 方は どなたですか",
    "× 私は 李さんです　○ 私は 李です"
  ]
}

// 块5 单元测 — L2/L3 可写「第4课后测」；L4 写 U1 全测范围
```
