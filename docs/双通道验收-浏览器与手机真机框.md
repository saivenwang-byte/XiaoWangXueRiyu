## 0b. 铁律（每一次刷星 / 修改）

1. **最终呈现**必须在 **真实手机逻辑尺寸** 的壳内，显示 **小程序 WebView 真实使用界面**（390×844 外壳；内容区 **753×390**；`miniappPreview=1`）。
2. **每一次**改 UI、刷星、调样式后，交付前须在该真机框内看到 **本轮最新保存** 的效果。
3. **Cursor 开发**：用 **Simple Browser** 打开真机预览页并 **改码即刷**（保存 css/js/html 后 web-view **刷新一次**，非盲轮询），不得只靠改代码不目视。

| 用途 | 地址 / 命令 |
|------|-------------|
| **Cursor 持续预览（首选）** | `http://127.0.0.1:8765/cursor-miniapp-phone.html?live=1` |
| 一键打开 | 双击 **`Cursor真机持续预览.bat`** |
| 双通道 | **`打开双通道预览.bat`**（全宽 + 真机框 live） |

**Cursor 操作**：`Ctrl+Shift+P` → `Simple Browser: Show` → 粘贴上表 URL → 侧边固定；须 **`重启本地服务.bat`**（`local-preview-server.py --watch`）；保存后 **一次** 刷新 iframe。Agent 收尾可 `python scripts/preview-notify-reload.py`。无 watch API 时兜底约 8 秒定时刷（可关「改码即刷」或 `?live=0`）。

## 2. 「持续更新」纪律（改 UI / 数据 / JS 时）

1. **先** 确保 `8765` 已起（`打开双通道预览.bat` 或 `重启本地服务.bat`）。
2. **开发全程** 保持 A、B 不关；**B 推荐 `?live=1` 改码即刷**：
   - A：`Ctrl+F5` 强刷，或确认 URL 中 `?v=` 与 `js/share-wechat.js` 的 `CACHE_VER` 一致；
   - B：保存 `css/`、`js/`、`html` 后 **自动刷新 web-view 一次**；亦可手动点 **「刷新 WebView」**；壳内可手指滚动/点按（触摸区与真机一致）。
3. **禁止** 仅用 Cursor 内嵌全宽预览、或只改完不刷新就声称「已修好」。
4. **禁止** 用 GitHub 公网链接在本机代替 A/B 做改码验收（公网仅用于 push 后抽样）。

## 4. 最终检查（交付用户 / 声称修好前）

在 **`发布前自检.bat` 全 [OK]** 之后，还必须完成 **双通道目视**：

| # | 检查项 | 通过标准 |
|---|--------|----------|
| 1 | 通道 A | 本次改动页面在 `index.html?v=` 下可见、无控制台阻断错误 |
| 2 | 通道 B | 同一改动在 **390×844 框内** 可见；底栏不挡内容；无横向溢出 |
| 3 | 小程序路径 | B 内 iframe 加载的 URL 与 `japanese_learning_miniapp/config/h5-url.js` 公网源一致（本地为 `127.0.0.1:8765` 同源） |

**开发者满配**（四关、彩蛋、testcard）仍按 [开发者验收-SOP.md](./开发者验收-SOP.md) 用 `dev-phone-preview.html`；与 B **并列**，不互相替代。
