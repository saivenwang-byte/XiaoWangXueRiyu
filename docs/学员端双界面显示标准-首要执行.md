## 一、最终成果的两种显示界面

标日 H5 学员端**交付完成**时，必须同时满足以下两种显示界面（缺一不可）：

| # | 界面 | 用途 | 唯一正确入口 | 通过标准 |
|---|------|------|----------------|----------|
| **1** | **本地 Cursor 预览器** | 开发、改 UI、交付前目视验收 | `http://127.0.0.1:8765/cursor-miniapp-phone.html?live=1`（一键：`Cursor真机持续预览.bat` / `打开双通道预览.bat`） | 在 **§二** 锁定尺寸壳内操作、切换各页仍符合 §二 |
| **2** | **可转发微信链接** | 学员真机学习、人传人转发 | `https://saivenwang-byte.github.io/XiaoWangXueRiyu/index.html?v={CACHE_VER}`（见 `js/share-wechat.js` · `js/public-url.config.js`） | 微信内打开与本地真机框 **同一套 H5**；`?v=` 与 `CACHE_VER` 一致 |

**说明**

- 界面 **1** 不是桌面全宽浏览器代替手机；须走 **390×844 真机逻辑壳**（见 §二）。
- 界面 **2** 不是 `localhost` / `file://` / 同 WiFi IP；见 `.cursor/rules/production-netlify.mdc`。
- 日常开发可另开 **通道 A**（全宽 `index.html?v=`）对照数据与链接，但**不得**用 A 单独声称「最终 UI 已验收」——终检须 **1 + 2**（详见 [双通道验收-浏览器与手机真机框.md](./双通道验收-浏览器与手机真机框.md)）。

## 二、Cursor 浏览器 · 锁定显示（铁律 · UI 设计规范补充）

在 **Cursor 内置浏览器 / Simple Browser** 中验收界面 **1** 时，必须满足：

| # | 要求 | 含义 | 真源 / 实现 |
|---|------|------|-------------|
| **L1** | **真实手机屏幕尺寸** | 外壳固定 **390×844** 逻辑像素（iPhone 14 / 微信开发者工具同规格）；内容区 **753×390**（扣状态栏 47px + 导航条 44px） | `cursor-miniapp-phone.html` `:root --phone-w` / `--phone-h` |
| **L2** | **在真实手机屏幕中操作各 UI** | 在壳内 iframe（`miniappPreview=1`）中点按、滚动、切换底栏 Tab、进课、五关、笔记、入門；体验与微信 WebView **同一视口与触摸区** | H5 加载 `css/miniapp-webview.css`；`index.html?…&miniappPreview=1` |
| **L3** | **切换界面时尺寸不变** | 从开机 / 地图 / 课内 / 笔记 / 注音（`intro.html`）任意切换，**外壳宽高不得随页面变化**；仅 **壳内 web-view 内容** 滚动或换页 | 禁止把 Simple Browser 拉成全屏当验收；禁止 iframe 随内容撑高壳体 |

### L3 验收动作（每次改 UI 后建议走一遍）

1. 打开 `cursor-miniapp-phone.html?live=1`，确认外框仍为 **390×844**。
2. 壳内依次：**课文（地图）→ 笔记 → 注音 → 进某一课 → 五关 Tab 切换 → 返回地图**。
3. 全程：**外框像素尺寸不变**；底栏、顶栏比例与首屏一致；无整页横向溢出。

### Cursor 推荐操作

```text
Ctrl+Shift+P → Simple Browser: Show
→ 粘贴：http://127.0.0.1:8765/cursor-miniapp-phone.html?live=1
→ 侧边固定，勿拉成全屏替代真机框
```

须先 **`重启本地服务.bat`**（`local-preview-server.py --watch`）；保存 `css/` / `js/` / `html` 后 web-view **刷新一次**（`?live=1`）。
