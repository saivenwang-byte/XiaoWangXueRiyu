## 1. 画风（直接沿用绘本规范）

| 维度 | 绘本 | 四格条 |
|------|------|--------|
| 线条 | 手绘圆润、无锐角 | 全 6 单元统一 |
| 背景 | 季节纯色/渐变，无复杂纹理 | 对齐 `CURRICULUM_STAGE_THEMES` |
| 建筑 | 卡通抽象（鸟居、屋檐、灯笼…） | 机场/办公室/温泉/站台同规则 |
| 主人公 | グルミ（叶色、头顶花、注连绳随季变） | **每格可有グルミ**；见绘本附录状态表 |
| 其他人类 | 远景剪影或简笔 | 森島・佐藤等 **无独立人设稿** |
| 台词 | 短句、少女/观察口吻；可第三人称指「あの人（李）」 | 每格 ≤1 气泡 + 可选 1 行中文 |
| 豆知識小字 | 绘本底部旁白 | **不进 L2 四格**（留 L3 故事书） |
| 特效 | キラキラ、烟花等 | L2 克制；L3 可加强 |

脚本取向：`.txt` 少女向 pastel 为主；`.md` 观察笔记语气可作旁白参考。

## 4. 人才全景 = 全项目级 Soul（非仅漫画）

任何阶段（入門、目录、四关、マイ、彩蛋、文案）都用同一套思考习惯，**不是**把岗位表画进界面：

| 维度 | 在本项目的用法 |
|------|----------------|
| **领域池** | 先问「这一块属于哪类生活/能力域」，再落 UI 或一格画 |
| **三层** | 通用打底 → 协同联动 → 核心验收；对应 00 / 单元内 / 金星 |
| **作业法** | 固定顺序：看清全局 → 分块学 → 合成运用 → 可验证完成 |
| **金句锚定** | 每单元条带 1 句日文收藏名，好记、好截图 |
| **图例先行** | 剪影/图标/四格规则先定，再填内容 |

**Soul**：用全景习惯做决策，**不**增加「再给小李画一张脸」这类无效工序。
