## 1. 三条结果（不可删改）

| # | 学员侧结果 | 实现要点 |
|---|------------|----------|
| ① | **默认干净** | 每行只露：日文、读音、**一条**中文义；右侧固定 **🔊 🎤 ▶**。朗读时不叠字、不弹全屏挡表。 |
| ② | **延伸按需** | 有 `L1KnowledgeTips`（或等价 `extend`）才显示 **「延伸」**；**行内折叠**展开 1～3 句 + 可选「→ 文法/会話」链接。 |
| ③ | **注意要能看懂** | 淡黄行 + **「注意」** 角标；点 **▸** 或点 **「注意」** → 展开 **「注意 · 要点」**（中文说明为何标黄），**不是**标成延伸。 |

## 2. 标黄判定（何时 `special`）

**不是**每词都标黄。仅下列情况打标（可多标并存）：

| 角标 | 含义 | 典型 |
|------|------|------|
| **注意** | 与「普通名词 + です」不同：称谓、礼貌、读法、易混 | 先生、あなた、父/母、はい/いいえ、～さん |
| **延伸** | 有延伸阅读文案 | `l1-knowledge-tips.js` 该 `vocab.id` 有条目 |
| **活用** | 有词形变化表（动词/形容词等） | `conjugation` 数据存在时显示；第1課多数词可无 |

**禁止**：无原因标黄；用全屏 modal 代替行内折叠；单词关加会話用 **✓ 评分** 第四键。

## 3. 视觉规范（对齐五十音，全课一致）

> **平面轻量铁律**：[产品设计规范-平面轻量UI.md](./产品设计规范-平面轻量UI.md)  
> **密度与统筹**：[项目知识库-U1L1-MVP-UI统筹审查.md](./项目知识库-U1L1-MVP-UI统筹审查.md) §2

- **折叠态尽量窄**：每词 **白底 + 1px 细框** `#e0e0e0`；行距 **4px**；字段横排：**序 → ▸ → 日文 → かな·语调·词性 → 角标 → 中文义 → 🔊🎤▶**（优先一行，极窄屏中文可第二行）。
- **二态底色（禁止渐变）**：普通行 **白**；**注意** 词整行 **淡黄** `#fff8e1` + 左 **3px** `#ffe082`。**延伸展开区用浅灰** `#f5f5f5`，**勿**与注意黄相近（防视网膜误判）。
- **角标**：行内小 pill：`注意` / `延伸` / `活用`（9px 紧凑，无阴影）。
- **展开 ▸**：彩色 **无底框三角**（`hyouga-glyphs` · 単語关蓝色），非黄底方钮。
- **页底**：三块 `<details>` 默认折叠、手风琴单开；头 **白底** + 蓝标题，与「注意黄」分工。

样式实现：`.l1-vocab-warn` / `.has-vocab-expand` + `lesson-1-flow.js`；**勿** `app.js` 第二套单词表。
